\section{Použité vývojové prostriedky}
V~tejto kapitole si priblížime technológie
použité pri vývoji tejto diplomovej práce.

\subsection{Softvér}

\subsubsection{JavaScript}
JavaScript je skriptovací programovací jazyk
a~základný stavebný kameň pri vývoji aplikácií
pre Zepp OS -- operačný systém používaný
v~zariadeniach Amazfit, ako sú napríklad
hodinky Amazfit T-Rex 3. Spolu s~HTML a~CSS
tvorí tretiu vrstvu štandardných webových
technológií. Umožňuje implementovať interaktívne
funkcie na webových stránkach, ako sú dynamické
aktualizácie obsahu, interaktívne mapy
či animovaná grafika.

\subsubsection{Node.js}
Node.js je prostredie na beh JavaScriptu
mimo webového prehliadača, predovšetkým
na serverovej strane. Je postavený na engine 
V8 od spoločnosti Google -- tom istom, ktorý poháňa 
prehliadač Chrome.
Funguje asynchrónne a~riadi sa udalosťami,
čo mu umožňuje spracovávať viacero požiadaviek
naraz bez čakania na dokončenie predchádzajúcej
úlohy. Vďaka tomu je vhodný na tvorbu webových
serverov, real-time aplikácií a~\Gls{api}
rozhraní -- vrátane Zepp OS \Gls{api}.
Disponuje rozsiahlym ekosystémom knižníc
prostredníctvom \Gls{npm}. \cite{Node.js-definition}

\subsubsection{Node Package Manager}
\Gls{npm} je správca
balíčkov pre JavaScript a~Node.js. Umožňuje
sťahovať, inštalovať a~spravovať knižnice od
iných vývojárov pomocou príkazového riadku
(napr. \texttt{npm i -g @zeppos/zeus-cli}). Zároveň
rieši verzie balíčkov a~ich závislosti, aby
všetko fungovalo bez konfliktov.
\cite{npm-definition}

\subsubsection{Zepp OS Mini Program}
Aplikácie pre Zepp OS sa vyvíjajú vo forme
Mini Programov -- odľahčených aplikácií
písaných v~JavaScripte. Kompletný Mini Program
sa skladá z~troch logických
celkov \cite{zepp_architecture}:

\begin{itemize}
    \item \textbf{Device App} -- časť bežiaca
    priamo na hodinkách, zodpovedná za \Gls{ui},
    volania senzorov a~spracovanie dát
    na zariadení.
    \item \textbf{Side Service} -- služba
    bežiaca na mobilnom telefóne bez \Gls{ui}
    rozhrania, zabezpečujúca dátovú komunikáciu
    so zariadením.
    \item \textbf{Settings App} -- časť
    na mobilnej strane bežiaca v~Zepp aplikácii
    s~\Gls{ui}, slúžiaca ako sprievodná
    stránka nastavení.
\end{itemize}

Komunikácia medzi Device App a~Side Service
prebieha prostredníctvom \Gls{ble} (Bluetooth
Low Energy). Keďže natívne \Gls{ble} \Gls{api}
je pomerne nízkoúrovňové, Zepp OS poskytuje
pomocnú knižnicu MessageBuilder, ktorá
abstrahuje odosielanie a~príjem správ medzi
oboma stranami do jednoduchého
rozhrania. \cite{zepp_bluetooth}

V~našej práci je Device App zodpovedná za 
snímanie senzorových dát z~gyroskopu a~akcelerometra. 
Tieto dáta sú následne posielané cez \Gls{ble} do Side 
Service a odtiaľ do Android aplikácie, kde sú 
spracovávané a~analyzované a dôjde k~rozpoznaniu gesta. Na 
základe rozpoznaného gesta sa aktualizuje stav 
zápasu a skóre. Skóre je následne odosielané späť 
do Device App a zobrazené na hodinkách.
Rozpoznané gesto a~aktuálny stav skóre sú potom odosielané
cez \Gls{ble} do Android aplikácie, ktorá slúži ako hlavné zobrazovacie
rozhranie pre obsluhu zápasu.

\subsubsection{Zepp OS Sensor API}
Zepp OS poskytuje prístup k~hardvérovým senzorom hodiniek
prostredníctvom modulu \texttt{@zos/sensor}. Akcelerometer
a~gyroskop sú dostupné od API\_LEVEL 3.0. \cite{zepp_sensor_acc} \cite{zepp_sensor_gyro}

Akcelerometer meria zrýchlenie zariadenia pozdĺž troch
ortogonálnych osí (X, Y, Z), pričom osi X a~Y sú rovnobežné
s~displejom a~os Z~je kolmá na displej. Gyroskop meria uhlovú
rýchlosť rotácie tiež v~troch osiach. Oba senzory fungujú
na princípe registrácie callback funkcie, ktorá je zavolaná
pri každej novej vzorke dát \cite{zepp_sensor_acc} \cite{zepp_sensor_gyro}:

\begin{lstlisting}[caption=Príklad inicializácie akcelerometra v~Zepp OS]
import { Accelerometer, FREQ_MODE_NORMAL } from '@zos/sensor'

const accelerometer = new Accelerometer()
accelerometer.onChange(() => {
    console.log(accelerometer.getCurrent())
})
accelerometer.setFreqMode(FREQ_MODE_NORMAL)
accelerometer.start()
\end{lstlisting}

\subsubsection{Zeus CLI}
Zeus \Gls{cli} je oficiálny nástroj terminálu pre vývoj
Zepp OS aplikácií, distribuovaný ako npm balík
\texttt{@zeppos/zeus-cli}. Vyžaduje Node.js vo verzii 14.0.0
alebo vyššej. \cite{zepp_cli}

Nástroj pokrýva celý vývojový cyklus aplikácie prostredníctvom
sady príkazov:

\begin{itemize}
    \item \texttt{zeus create} -- interaktívne vytvorenie nového projektu
    \item \texttt{zeus dev} -- kompilácia s~náhľadom v~simulátore
    a~automatickým obnovením pri zmene kódu
    \item \texttt{zeus preview} -- kompilácia pre reálne zariadenie
    s~vygenerovaním QR kódu na inštaláciu cez vývojárske nastavenia Zepp mobilnej aplikácie
    \item \texttt{zeus build} -- zostavenie finálneho inštalačného
    balíčka aplikácie
    \item \texttt{zeus bridge} -- Developer Bridge režim pre živé
    ladenie bežiacej aplikácie na zariadení
\end{itemize}

\subsubsection{Kotlin}
Kotlin je open-source, staticky typovaný programovací jazyk, ktorý
podporuje objektovo-orientované aj funkcionálne programovanie.
Vyvíja ho spoločnosť JetBrains od roku 2011, pričom prvé stabilné
vydanie vyšlo v~roku 2016. Od roku 2019 je Kotlin preferovaným
jazykom Googlu pre vývoj Android aplikácií. \cite{kotlin_android}

Kotlin je plne interoperabilný s~jazykom Java, čo znamená, že
v~jednom projekte môžu koexistovať oba jazyky a~navzájom sa volať.
Oproti Jave prináša Kotlin výrazne stručnejšiu syntax, zabudovanú
ochranu pred null pointer výnimkami a~podporu pre korutiny
(\textit{coroutines}), ktoré zjednodušujú prácu s~asynchrónnym
kódom. \cite{kotlin_overview}

\subsubsection{Android Studio}
Android Studio je oficiálne integrované vývojové prostredie (\gls{ide})
pre vývoj Android aplikácií. Je postavené na platforme IntelliJ
IDEA od spoločnosti JetBrains a~bolo predstavené na konferencii
Google I/O~v~roku 2013 ako náhrada za dovtedajší Eclipse ADT.
\cite{android_studio_intro}

Medzi hlavné funkcie Android Studia patria:
\begin{itemize}
    \item inteligentný editor kódu s~dopĺňaním pre Kotlin a~Javu
    \item vstavaný emulátor Android zariadení na testovanie
    bez fyzického hardvéru
    \item Layout Editor na vizuálny návrh UI rozhraní
    \item Gradle-based build systém na správu závislostí
    a~zostavenie aplikácie
    \item nástroje na profilovanie výkonu a~ladenie aplikácií
    (Logcat, Profiler)
\end{itemize}

Android Studio je dostupné bezplatne pod licenciou Apache 2.0
a~je kompatibilné so systémami Windows, macOS aj Linux.
\cite{android_studio_intro}


\subsubsection{Python}
Python je interpretovaný, vysokoúrovňový
programovací jazyk s~dynamickým typovaním.
Kladie dôraz na čitateľnosť a~jednoduchosť
syntaxe, čo znižuje náklady na údržbu kódu.
Podporuje modulárny prístup k~vývoju
a~opätovné použitie kódu prostredníctvom
balíčkov a~modulov. V~našej práci sme ho
použili na spracovanie výstupov z~gyroskopu
a~akcelerometra, tvorbu neurónovej siete na
rozpoznávanie gest a~vizualizáciu nameraných
dát prostredníctvom grafov.
\cite{python-definition}

\subsubsection{Pandas}
Pandas je open-source knižnica v~Pythone určená
na manipuláciu a~analýzu štruktúrovaných dát.
Jej hlavnými dátovými typmi sú Series
(jednorozmerné pole s~indexmi) a~DataFrame
(dvojrozmerná tabuľka podobná Excelu).
Umožňuje čítať dáta z~rôznych formátov
(\Gls{csv}, Excel, \Gls{json}, \Gls{sql}),
čistiť ich, filtrovať, zoskupovať
a~vykonávať štatistické výpočty. V~našej práci
ju používame na spracovanie výstupných dát
vo formáte \Gls{csv}. \cite{pandas-definition}

\subsubsection{Knižnica Matplotlib}
Matplotlib je open-source knižnica v~Pythone
slúžiaca na vizualizáciu dát. Umožňuje
vytvárať rôzne typy grafov -- od čiarových
a~stĺpcových cez bodové až po 3D vizualizácie.
Vznikla v~roku 2003 a~najčastejšie sa používa
prostredníctvom modulu pyplot, ktorý poskytuje
intuitívne príkazy podobné prostrediu MATLAB.
